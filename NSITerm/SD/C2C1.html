    C2C1 – Structures de données linéaires : piles et files⚓︎

    Introduction⚓︎

    Les structures de données linéaires permettent de regrouper des données et de définir des opérations de base pour les manipuler.
    Elles sont dites “linéaires” car les éléments sont organisés les uns à la suite des autres.

    Dans le programme NSI, cette partie s’inscrit dans le thème « Structures de données ».
    Pour mieux comprendre la démarche, on distingue ici trois niveaux : - la description abstraite du comportement attendu (souvent appelée TAD en algorithmique), - la structure de données concrète utilisée en mémoire, - l’implémentation de ces opérations en code.

    Parmi les structures linéaires les plus courantes, on trouve :
    - la pile (LIFO),
    - la file (FIFO),
    - la liste.

    🟦 TAD, Structure de données, Implémentation : bien faire la distinction

    Lorsqu’on étudie les structures linéaires (listes, piles, files…), il est essentiel de distinguer :

    • 🧠 Le TAD (Type Abstrait de Données)
      → C’est une abstraction. Elle définit ce qu’on peut faire (opérations disponibles) et leur signification.
      → Exemple : le TAD Pile définit creer, empiler, depiler, sommet, est_vide.
      → Le TAD ne dit rien sur la représentation en mémoire.

    • 🧱 La structure de données
      → C’est une représentation concrète utilisée pour implémenter le TAD (tableau, liste chaînée, etc.).
      → Un même TAD peut avoir plusieurs structures différentes.

    • 🧰 L’implémentation
      → C’est le code qui réalise les opérations du TAD avec une structure donnée.

    📌 Un même TAD peut être implémenté par plusieurs structures différentes, et une même structure peut servir à plusieurs TAD.

    La structure de données Pile⚓︎

    Définition⚓︎

    Une pile est une structure de données linéaire dans laquelle les insertions et les suppressions se font à une extrémité unique : le sommet.
    Elle suit la règle LIFO (Last In, First Out).

    Interface de la pile⚓︎

    • creer(capacite) : créer une pile vide bornée de taille maximale capacite
    • empiler(p, x) : ajouter x au sommet de la pile p
    • depiler(p) : retirer et renvoyer l’élément au sommet
    • sommet(p) : accéder à l’élément au sommet sans le retirer
    • est_pile_vide(p) : tester si la pile est vide

    Implémentation : pile bornée avec pile[0] comme indice du sommet⚓︎

    On choisit ici une structure simple :
    - un tableau de taille capacite + 1,
    - pile[0] contient l’indice du sommet (0 si pile vide),
    - les éléments empilés sont stockés à partir de l’indice 1.

    def creer_pile(capacite):
    return [0] + [None] * capacite

def est_pile_vide(p):
    return p[0] == 0

def empiler(p, x):
    assert p[0] < len(p) - 1, "Erreur pile pleine"
    p[0] += 1
    p[p[0]] = x

def depiler(p):
    assert not est_pile_vide(p), "Erreur pile vide"
    x = p[p[0]]
    p[0] -= 1
    return x

def sommet_pile(p):
    assert not est_pile_vide(p), "Erreur pile vide"
    return p[p[0]]

print(depiler(p))      # 10

    Toutes les opérations sont en temps constant \(\mathcal{O}(1)\).

    La structure de données File⚓︎

    Définition⚓︎

    Une file est une structure de données linéaire dans laquelle les insertions se font à une extrémité (l’arrière) et les suppressions à l’autre (l’avant).
    Elle suit la règle FIFO (First In, First Out).

    Interface de la file⚓︎

    • creer() : créer une file vide
    • enfiler(f, x) : ajouter x à l’arrière
    • defiler(f) : retirer et renvoyer l’élément en tête
    • est_file_vide(f) : tester si la file est vide

    1. Implémentation triviale avec une liste Python⚓︎

    On peut utiliser directement les méthodes natives append et pop(0) de Python :

    def creer_file():
    return []

def est_file_vide(f):
    return len(f) == 0

def enfiler(f, x):
    f.append(x)

def defiler(f):
    assert not est_file_vide(f), "Erreur file vide"
    return f.pop(0)

    Opération Complexité
    enfiler \(\mathcal{O}(1)\) amorti
    defiler \(\mathcal{O}(n)\) (décalage)

    👉 Cette version est très simple à comprendre, mais defiler est de complexité en temps linéaire car Python doit décaler tous les éléments de la liste.

    2. Implémentation avec deux piles⚓︎

    Le programme NSI demande de présenter la construction d’une file à partir de deux piles.

    📌 Principe⚓︎

    • enfiler(x) → empiler x sur la pile entree.
    • defiler() :
    • si la pile sortie est vide → transférer tous les éléments d’entree vers sortie (inverser l’ordre),
    • dépiler le sommet de sortie.

    Chaque élément est transféré au plus une fois → complexité amortie en O(1).

    def creer_file_piles(capacite):
    entree = creer_pile(capacite)
    sortie = creer_pile(capacite)
    return (entree, sortie)

def est_vide_file_piles(f):
    entree, sortie = f
    return est_pile_vide(entree) and est_pile_vide(sortie)

def enfiler_piles(f, x):
    entree, sortie = f
    empiler(entree, x)

def defiler_piles(f):
    entree, sortie = f
    if est_pile_vide(sortie):
        while not est_pile_vide(entree):
            empiler(sortie, depiler(entree))
    assert not est_pile_vide(sortie)
    return depiler(sortie)

print(defiler_piles(f))  # D
    Opération Complexité amortie
    enfiler \(\mathcal{O}(1)\))
    defiler \(\mathcal{O}(1)\) amortie \(\mathcal{O}(n)\) si transfert ponctuel)

    👉 Cette implémentation est une belle illustration de la distinction entre TAD et structure : la structure est ici composée de deux piles au lieu d’un tableau ou d’une liste.

    Bilan⚓︎

    • La pile et la file sont deux structures fondamentales, dont on peut décrire le comportement abstrait puis choisir différentes structures concrètes pour les réaliser :
    • Pour la pile : tableau simple (pile[0] = sommet),
    • Pour la file : liste Python (simple) ou deux piles (efficace).
    • Ces exemples montrent bien qu’un même comportement abstrait peut être implémenté de plusieurs manières différentes.
